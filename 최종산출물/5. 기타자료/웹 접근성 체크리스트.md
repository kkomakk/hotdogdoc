- [ ] **2.** 시맨틱 태그 구조(`main`, `nav`, `section`, `article`, `header`, `footer` 등)를 지켰는가?
- [ ] **3.** 페이지 이동은 `<a>`, 동작(제출·토글 등) 실행은 `<button>`으로 구분했는가?
- [ ] **4.** `<label>`의 `for`와 `<input>`의 `id`가 1:1로 정확히 매칭되었는가? (해당 페이지에 폼이 없으면 N/A)
- [ ] **5.** Tab 포커스 순서가 콘텐츠의 시각적·논리적 순서와 맞는가?
- [ ] **6.** 모든 이미지에 적절한 `alt`가 있는가? (장식용은 `alt=""` 또는 `role="presentation"`)
- [ ] **7.** HTML 루트(`<html lang="...">`)에 기본 언어를 명시했는가?
- [ ] **8.** `<!DOCTYPE html>` 선언이 있는가?
- [ ] **9.** 아이콘·메뉴 등에 `aria-label` 등으로 이름/역할이 전달되는가?
- [ ] **10.** 복잡한 제스처 없이, 클릭(또는 키보드)만으로 동일 기능을 수행할 수 있는가?
- [ ] **11.** 화면에 보이지 않는 요소(숨김 영역)에 포커스가 가지 않도록 처리했는가? (`tabindex`, `aria-hidden` 등)

## [P1] 중요

- [ ] **1.** 모든 에러 문구가 **어떤 필드·어떤 문제**와 연결되는지 식별 가능한가? (`aria-describedby`, 인라인 에러 등)
- [ ] **2.** 포커스 링/외곽선이 시각적으로 뚜렷한가? (다크 모드 포함)
- [ ] **3.** 모달(팝업)이 열리면 포커스가 모달 안에 가두어지는가? (Focus trap)
- [ ] **4.** 아코디언 등에서 `aria-expanded`, `aria-controls` 등이 적절한가?
- [ ] **5.** 모달이 열릴 때 포커스가 모달로 이동하고, 닫힐 때 트리거 버튼 등으로 돌아가는가?
- [ ] **6.** 필수 입력에 `aria-required` 또는 명확한 필수 표시가 있는가?
- [ ] **7.** `aria-describedby` 등으로 보조 설명이 스크린 리더에 전달되는가?
- [ ] **8.** 장식용 아이콘·중복 텍스트에 `aria-hidden` 등이 적절한가?
- [ ] **9.** 자동 재생(슬라이드·영상 등)은 일시정지·중지 수단이 있는가?
- [ ] **10.** 전체 화면 전환 없이 폼 제출 성공/데이터 로드 시, 스크린 리더가 결과를 알 수 있는가? (`role="status"`, `aria-live` 등)
- [ ] **11.** 제스처·드래그 등을 쓴 경우, 단일 포인터(클릭/탭)로도 동일 목표에 도달할 수 있는가?
- [ ] **12.** 링크·버튼·탭 등 터치/클릭 영역이 너무 작지 않은가? (권장 최소 약 44×44px 등 팀 기준)
- [ ] **13.** 글자 크기 조정 UI가 있으며, 눈에 잘 띄는가? (해당 서비스 정책 시)
- [ ] **14.** 글자 크기를 **최대 약 400%**까지 키웠을 때 텍스트 겹침·잘림이 없는가?
- [ ] **15.** 400% 확대 시 레이아웃이 깨지지 않고, 필요 시 반응형으로 전환되는가?

- [ ] **6.** 페이지네이션 등에서 현재 페이지가 색만이 아니라 형태(테두리·배경 등)로도 구분되는가?

- [ ] **9.** 목록 콘텐츠에 `<ul>`/`<ol>`과 `<li>`를 적절히 사용했는가?
